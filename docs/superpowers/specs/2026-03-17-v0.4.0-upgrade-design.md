# v0.4.0 Upgrade Design

> Date: 2026-03-17
> Target release: v0.4.0
> Branch strategy: Single `release/v0.4.0` branch, sequential batches, commit between each step

## Overview

Upgrade all major dependencies in a single release. No downstream consumers exist, so breaking changes are unconstrained. The 928-test suite is the primary safety net between each step.

**Validation gate between every step:**
```bash
npm run build && npm test && npm run typecheck && npm run lint
```

## Upgrade Order

| Step | Batch | Rationale |
|------|-------|-----------|
| 0 | Storybook patches | Zero-risk warmup |
| 1 | Vite 8 + Vitest 4 + plugin-react 6 | Build toolchain — everything else depends on it |
| 2 | Tailwind CSS 4 + tailwind-merge 3 | Styling system rework before Storybook picks up new config |
| 3 | React 19 | Runtime — drop React 18 support |
| 4 | Storybook 10 | Dev tooling — last because it depends on final Vite, Tailwind, and React versions |
| 5 | Cleanup | jsdom 29, remove stale deps, final audit, version bump |

## Step 0: Storybook Patches

The three core Storybook packages are currently at `^8.5.0` (not `^8.6.x`). Update their version ranges to `^8.6.18` to align with `@storybook/addon-a11y` which is already at `^8.6.18`:

```bash
npm install @storybook/react@^8.6.18 @storybook/react-vite@^8.6.18 storybook@^8.6.18 @storybook/addon-essentials@^8.6.18
```

Sanity-check Storybook dev server after install.

## Step 1: Vite 8 Batch

### Packages

| Package | Current | Target |
|---------|---------|--------|
| `vite` | ^6.0.0 | ^8.0.0 |
| `@vitejs/plugin-react` | ^4.3.0 | ^6.0.0 |
| `vitest` | ^3.0.0 | ^4.1.0 |
| `vitest-axe` | ^0.1.0 | latest compatible |
| `vite-plugin-dts` | ^4.3.0 | latest compatible |

Install all packages in a single `npm install` invocation to avoid intermediate peer dependency conflicts.

### Changes

- **Update `engines` field** — Vite 8 requires Node.js 20.19+. Change `"node": ">=18.0.0"` to `">=20.19.0"` in `package.json`.

- **Migrate `build.rollupOptions` to `build.rolldownOptions`** — Vite 8 replaces Rollup with Rolldown. The current `vite.config.ts` uses `build.rollupOptions` for externals (`react`, `react-dom`, `react/jsx-runtime`) and output globals. Migrate to `build.rolldownOptions` with equivalent config.

- **Audit for Babel usage** — `@vitejs/plugin-react` v6 replaces Babel with Oxc. Check for `.babelrc`, `babel.config.*`, or custom Babel plugins passed to the plugin. If found, either remove them or add `@rolldown/plugin-babel` as an opt-in. Currently the project uses `react()` with no options, so this is likely a no-op — but verify.

- **Verify `vitest-axe` compatibility** — this community package is used in `src/test-setup.ts`. If incompatible with Vitest 4, find an alternative or pin to a working version.

- **Update `vite.config.ts`** — beyond the rolldown migration, verify `__dirname` usage and `vite-plugin-dts` config still works.

- **Verify build output** — `dist/index.mjs`, `dist/index.cjs`, `dist/index.d.ts` should be present and correct.

### Risks

- `vite-plugin-dts` may not yet support Vite 8 — evaluate alternatives (`tsup`, direct `tsc` emit) or pin to compatible version
- `vitest-axe` may not be compatible with Vitest 4 — evaluate alternatives

## Step 2: Tailwind CSS 4 + tailwind-merge 3

This is the largest step. Tailwind 4 replaces JS config with CSS-first configuration.

### Packages

| Package | Current | Target |
|---------|---------|--------|
| `tailwindcss` | ^3.4.17 | ^4.2.0 |
| `tailwind-merge` | ^2.6.0 | ^3.0.0 |
| `@tailwindcss/vite` | — | ^4.2.0 (new) |
| `autoprefixer` | ^10.4.20 | **remove** |
| `postcss` | ^8.5.0 | **remove** |

### Changes

1. **Delete `tailwind.config.ts`** — all theme config moves into CSS

2. **Delete `postcss.config.js`** — Tailwind 4 runs as a Vite plugin, PostCSS is no longer needed. Remove both `postcss` and `autoprefixer` from devDependencies.

3. **Add `@tailwindcss/vite` to both config files** — install `@tailwindcss/vite` and add it as a plugin in **both** `vite.config.ts` (for library build) and `vitest.config.ts` (for test rendering). Without this, tests that render components with Tailwind classes will not process styles correctly.

4. **Rewrite `src/styles/tokens.css`** — use `@theme inline` (not plain `@theme`) because all color values reference CSS custom properties (`var(--primary)`, etc.) for runtime theme switching. Plain `@theme` would resolve variables at build time, breaking light/dark/high-contrast theming. The `@theme inline` block defines:
   - Colors: border, primary, secondary, destructive, muted, accent, card, subtle, success, warning, error, info, severe
   - Border radius scale: none, sm, DEFAULT, md, lg, xl, 2xl, 3xl, full
   - Box shadows: 2, 4, 8, 16, 28, 64
   - Font sizes with line-height/weight: caption-2 through display
   - Font family: sans (Segoe UI stack)
   - Animations: wave-spin, wave-pulse, wave-indeterminate

5. **Keep `src/styles/animations.css` as a separate file** — this file contains `@keyframes` definitions plus a `prefers-reduced-motion` media query (WCAG 2.1 SC 2.3.3 compliance). Keyframes are not part of `@theme` in Tailwind 4. Keep this file imported from `globals.css` as-is. The animation *names* are registered in `@theme inline` so Tailwind generates the utility classes, but the actual `@keyframes` rules remain in `animations.css`.

6. **Update `src/styles/globals.css`** — replace `@tailwind base/components/utilities` with `@import "tailwindcss"` syntax. Keep `@import './tokens.css'` and `@import './animations.css'`. Retain the existing `@layer base { ... }` block unchanged (sets global font-family, color, background-color, and border-color defaults).

7. **Update `tailwind-merge` to v3** — audit `cn()` in `src/lib/` to ensure merge behavior is correct with Tailwind 4 class names. Note: `tailwind-merge` is a runtime dependency (in `dependencies`, not `devDependencies`), so this is a semver-visible change for any future consumers. Acceptable since no consumers exist yet.

8. **Content detection** — Tailwind 4 auto-detects content (no `content: [...]` config needed). Verify it picks up `src/` and `stories/`.

9. **Class name audit** — run build, visually spot-check Storybook stories (Button, Card, Dialog, Table, Toast) across light/dark themes.

### Validation

Standard gate plus: launch Storybook, visually check component sampling across light/dark themes.

## Step 3: React 19

### Packages

| Package | Current | Target |
|---------|---------|--------|
| `react` | ^18.3.1 | ^19.2.0 |
| `react-dom` | ^18.3.1 | ^19.2.0 |
| `@types/react` | ^18.3.0 | ^19.0.0 |
| `@types/react-dom` | ^18.3.0 | ^19.0.0 |
| `@testing-library/react` | ^16.1.0 | latest compatible |

### Changes

1. **Update `peerDependencies`** — change from `^18.0.0 || ^19.0.0` to `^19.0.0`

2. **Remove `forwardRef` wrappers (optional)** — React 19 passes `ref` as a regular prop. All 65 components use `React.forwardRef`. These *can* be simplified to regular function components accepting `ref` in props. This is a large mechanical change and is **optional** — `forwardRef` still works in React 19 (deprecated but functional). Recommended to do it now since we have no consumers and it simplifies the codebase, but can be deferred if it creates too much churn.

3. **Remove deprecated APIs** — audit for `defaultProps` on function components, string refs, legacy context APIs

4. **Update test setup** — `act()` can be imported from `react` directly instead of `react-dom/test-utils`. Check `@testing-library/react` compatibility.

5. **Type changes** — `@types/react` 19 removes some deprecated types. Run typecheck and fix.

6. **Verify compound component patterns** — `Object.assign` pattern (e.g., `Card.Header`) must work correctly with new ref handling.

### Validation

Standard gate. Expect some test fixes for act/ref changes.

## Step 4: Storybook 10

### Packages

| Package | Current | Target |
|---------|---------|--------|
| `storybook` | ^8.6.18 | ^10.2.0 |
| `@storybook/react` | ^8.6.18 | ^10.2.0 |
| `@storybook/react-vite` | ^8.6.18 | ^10.2.0 |
| `@storybook/addon-essentials` | ^8.6.18 | ^10.2.0 |
| `@storybook/addon-a11y` | ^8.6.18 | ^10.2.0 |

### Changes

1. **Run Storybook migration CLI** — `npx storybook@latest upgrade` as starting point

2. **ESM-only constraint** — Storybook 10 is ESM-only. Audit `.storybook/main.ts`, `.storybook/preview.tsx`, and all story files for any `require()` calls or CJS patterns. Convert to ESM imports. Currently `main.ts` and `preview.tsx` use ESM imports, so this is likely a no-op — but verify after migration CLI runs.

3. **Config migration** — update `.storybook/main.ts` and `.storybook/preview.tsx` per new format

4. **Story format** — verify all stories work under new CSF format

5. **Addon compatibility** — confirm `addon-a11y` and `addon-essentials` have Storybook 10 versions. Drop and replace any that don't.

6. **Vite integration** — verify `@storybook/react-vite` works with Vite 8

### Validation

`npm run dev` launches Storybook, all stories render, a11y addon works, `npm run build-storybook` succeeds.

## Step 5: Cleanup

- **jsdom** — upgrade from 25 to 29
- **Lock file** — clean `npm install` from scratch, verify no peer dep warnings
- **Final validation** — `npm run build && npm test && npm run typecheck && npm run lint && npm run build-storybook`
- **Version bump** — update `package.json` version to `0.4.0`

## Out of Scope

- Publishing to npm
- Migration guides (no consumers)
- CI/CD pipeline changes
- New component development
